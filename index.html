IoT Air Quality & Environmental Monitoring Device

Project Overview

This wireless IoT device is built around an ESP32 microcontroller. It periodically collects air quality and environmental data from multiple sensors, formats the readings into a JSON payload, and sends it to a cloud API endpoint every 10 minutes over Wi-Fi.

Data was collected continuously for 2 days. The charts below represent only a small portion of the dataset to provide a clear visualization of trends and measurements. They are not real-time.

System Overview

  • BME280: Measures Temperature (°C), Humidity (%), Pressure (hPa), and Altitude (m)
  • SCD30: Measures CO₂ concentration (ppm) with automatic self-calibration
  • SPS30: Measures Particulate Matter (PM1.0–PM10) and particle counts (NC0.5–NC10) with data validation via CRC-8
  • SGP40: Measures VOC raw values and computes the TVOC index based on environmental conditions

Key Features

  • Periodic data collection every 10 minutes with autonomous operation
  • Multi-sensor integration into a single JSON payload for structured cloud transmission
  • Reliable Wi-Fi communication with automatic reconnection if disconnected
  • Data validation and error handling to ensure accurate readings, including flagging invalid SPS30 data

Technology Stack

  • Hardware: ESP32, BME280, SCD30, SPS30, SGP40
  • Software: Arduino IDE (C++), JSON formatting for payload construction
  • Communication: HTTP POST over Wi-Fi to cloud API endpoint

Outcome
